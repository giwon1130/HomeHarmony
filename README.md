# HomeHarmony

아파트 선택 과정에서 입지와 생활 편의 요소를 함께 비교할 수 있도록 돕는 주거 선택 지원 프로젝트다.

주거 선택을 단순 매물 조회가 아니라, 실제 생활 의사결정 문제로 풀어보는 것을 목표로 한 프론트엔드 MVP다.

## 프로젝트 목적
- 단순 매물 조회가 아니라 실제 거주 선택 기준에 가까운 정보를 한 번에 비교해보는 서비스를 만들고자 시작했다
- 교통, 편의시설, 생활 환경 같은 요소를 함께 보면서 의사결정을 돕는 방향을 목표로 했다

## 현재 구현 상태
현재 저장소는 React + TypeScript + Vite 기반의 실행 가능한 MVP로 구성되어 있다.

이 MVP에서 현재 동작하는 기능:
- 아파트 후보 목록 탐색
- 지역 / 예산 / 출퇴근 기준 필터링
- 교통 / 생활 / 교육 / 환경 점수 기반 정렬
- 상세 후보 보기
- 최대 3개 후보 비교
- Leaflet 기반 실제 지도 시각화
- 비교 후보와 선택 후보의 생활권 위치 탐색

## 구상한 핵심 기능
- 아파트 후보 비교
- 위치 기반 생활 편의 정보 조회
- 조건 기반 필터링
- 사용자 관점의 주거 선택 정보 정리

## 기술 스택
- Frontend: React, TypeScript, Vite
- Map: Leaflet, React Leaflet
- UI: CSS 기반 카드형 비교 화면
- Data: 현재 mock 데이터 기반
- 향후 방향: Spring Boot, PostgreSQL/PostGIS, 지도 기반 생활권 데이터 연계

## 핵심 화면
- 후보 카드 목록
- 조건 필터 패널
- 선택 후보 상세 패널
- 생활권 실제 지도
- 3개 후보 비교 테이블

## 로컬 실행
1. `npm install`
2. `npm run dev`
3. `npm run build`

## 앞으로 확장할 것
- 실제 아파트 및 생활권 데이터 연동
- 사용자 조건 저장 및 추천 기준 고도화
- 백엔드 API와 점수 산정 로직 분리

## 메모
지금 단계에서는 “주거 선택을 어떻게 비교 가능한 제품으로 만들지”를 보여주는 MVP에 가깝다.
이후 데이터 소스와 지도 기능이 붙으면 실제 서비스형 프로젝트로 확장할 수 있다.
